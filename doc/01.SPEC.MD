# Техзадание: Локальная система транскрибации диалогов из видеозаписей

**Версия**: 1.0  
**Дата**: 28.03.2026  
**Статус**: Проверка реализуемости и уточнение требований

---

## 1. Обзор проекта

### Задача
Создать инфраструктуру в контейнере ProxmoxVE для автоматической транскрибации диалогов из видеозаписей звонков с использованием **локальной модели Whisper** (без зависимости от онлайн API).

### Контекст
- **Успешный прототип**: `/v2t-decoder/test-example/` содержит работающий pipeline на онлайн Whisper API
- **Исходные данные**: видеозаписи формата `.mkv` / `.mp4` с двумя каналами аудио (левый/правый = два собеседника)
- **Желаемый результат**: автоматический диалог с таймлайнами в формате `.txt` и `.srt`

---

## 2. Анализ существующего кода

### 2.1 Компоненты в `test-example/`

#### A. `whisper_probe.py`
- **Назначение**: быстрая проба Whisper API для диагностики
- **Функции**:
  - Находит первый видеофайл (`.mp4` / `.mkv`)
  - Извлекает первую аудиодорожку в mono
  - Отправляет на онлайн Whisper с настройкой prompt
  - Сохраняет RAW JSON ответ
- **Статус**: Работает, но специализирован на диагностике

#### B. `extract_all_audio_variants.py`
- **Назначение**: экспорт аудиодорожек в разных вариантах (для анализа качества)
- **Функции**:
  - Использует `ffprobe` для анализа аудиопотоков
  - Создаёт для каждого потока 4 варианта: original, mono mix, left channel, right channel
  - Сохраняет в формат WAV
- **Статус**: Работает, но не используется в финальном pipeline

#### C. `transcribe_calls.py` ⭐ **ОСНОВНОЙ**
- **Назначение**: полный pipeline обработки видео + API транскрибация
- **Функции**:
  - Находит все видеофайлы (`.mp4` / `.mkv`)
  - Для каждого видео: извлекает левый (S1/FL) и правый (S2/FR) каналы в MP3
  - Отправляет оба на Whisper API
  - Собирает результаты в диалог с таймлайнами
  - Генерирует `.txt` и `.srt` файлы
  - **Важно**: добавляет offset 600 сек (10 минут) для каждого файла
- **Статус**: Полностью рабочий, использовалась в `test-example/`

#### D. `transcribe_from_extracted_audio.py`
- **Назначение**: альтернативный pipeline, работает с предварительно извлечёнными MP3
- **Функции**: аналогичны C, но требует предварительной подготовки файлов
- **Статус**: Рабочий, но менее практичный

## 3. Требования к системе (DEMO версия)

### 3.1 Входные данные
- **Формат**: `.mkv` / `.mp4`
- **Структура**: видеозапись звонка с двумя каналами аудио (стерео L/R)
- **Пользовательское действие**: загрузить файл и вызвать скрипт

```bash
python main.py input_video.mkv
```

### 3.2 Обработка (DEMO)
1. **Отрезание фрагмента**: первая **1 минута** видео
2. **Разбор на дорожки**: L → S1.mp3, R → S2.mp3
3. **Транскрибация**: отправка на Whisper
4. **Формирование диалога**: сборка в последовательный формат
5. **Вывод результата**: путь к файлу с диалогом (`.txt` / `.srt`)

**Время обработки**: ~10-30 сек (зависит от Whisper)

### 3.3 Выходные данные
- `dialog_timeline.txt` — диалог с таймлайнами
- `dialog_timeline.srt` — тот же диалог в формате SRT для плеера
- **(опционально)** сохранить LOG файл с метаданными обработки

---

## 4. Требования к PRODUCTION версии

### 4.1 Отличия от DEMO
- **Обработка**: полный видеофайл, а не 1 минута
- **Фрагментация**: автоматическое разбиение на 5-10 минутные куски
- **Стабильность таймлайнов**: корректировка `global_offset` между фрагментами

### 4.2 Добавочные требования
- **Обработка больших файлов** (10-100 часов видео)
- **Рестартуемость**: возможность продолжить с точки отказа
- **Параллелизм**: опция обработки нескольких фрагментов одновременно
- **Логирование**: подробные логи хода обработки

## 5. Ключевые вопросы для уточнения ❓

### 5.1 Выбор Whisper модели

**Текущий статус**: в прототипе используется онлайн `whisper-1` API OpenAI  
**Проблема**: требует интернета, платён, может быть недоступен

**Варианты**:
1. **`openai-whisper` (локальная Python модель)**
   - ✅ Работает локально, бесплатно
   - ❌ Медленнее онлайн API
   - ❌ Требует GPU/много RAM

2. **`faster-whisper` (оптимизированная версия)**
   - ✅ Быстрее на 4-10x, поддерживает quantization
   - ✅ Хороший компромисс между скоростью и качеством
   - ❓ Совместима с API OpenAI? Нужна адаптация?

3. **`Ollama` + Whisper контейнер**
   - ✅ Универсальное решение для ProxmoxVE
   - ✅ Легко масштабировать
   - ❌ Дополнительная инфра

**Решение**: рекомендую **`faster-whisper`** как оптимальный выбор
