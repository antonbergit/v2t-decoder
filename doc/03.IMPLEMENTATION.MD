## 🚧 Phase 3 (PRODUCTION) — текущий прогресс

Сделано в рамках старта Phase 3:

1. В `main.py` добавлено планирование временных чанков через `_build_chunks(...)`.
2. Production-режим теперь обрабатывает видео по временным окнам, а не одним проходом.
3. Для каждого чанка выполняются:
	- извлечение нужного диапазона аудио,
	- транскрибация `S1`/`S2`,
	- merge с глобальным `time offset`.
4. В JSON-лог добавлены:
	- стадия `chunk_plan`,
	- стадии `chunk_001`, `chunk_002`, ...
	- число чанков и их временные границы.
5. Сохранена текущая стратегия каналов:
	- default fixed mapping,
	- optional fast prescan.

Текущее состояние Phase 3:
- базовая chunked-обработка реализована;
- синтаксис, short smoke test и extended test (190s, 4 chunks) проходят;
- следующий шаг — расширить тестирование и зафиксировать production-этап отдельным коммитом.

## ✅ Phase 4 (Deployment) — курс на native release

Принято решение для целевого окружения:

1. Отказ от Docker в релизе (ограничения окружения: Docker build блокируется AppArmor/permission).
2. Docker-наработки сохранены в отдельной ветке `phase4-docker-poc`.
3. Основной путь релиза — native deployment (VM/LXC):
	- инструкция в [doc/05.DEPLOYMENT.MD](05.DEPLOYMENT.MD),
	- helper script [scripts/run_v2t.sh](../scripts/run_v2t.sh),

Статус Phase 4:
