## 🎯 Принятые архитектурные решения

### Выбор технологий

| Компонент | Выбор | Обоснование |
|-----------|-------|-------------|
| **Whisper модель** | `faster-whisper` (medium) | Локально, быстро, качественно |
| **Интеграция** | Прямая замена (рефакторизация) | Простая реализация для MVP |
| **Интерфейс** | YAML конфиг (argparse для --config) | Минимизация ошибок, повторяемость |
| **Логирование** | JSON структурированный | Удобство автоматического анализа |
| **Контейнер** | Docker (приоритет 1), VM (2) | Портабельность, управление зависимостями |
| **Ограничения** | Max 2 часа видео | Для MVP достаточно |

### Обработка граничных случаев

| Сценарий | Решение |
|----------|---------|
| Видео без аудио | ❌ Ошибка и выход |
| Моно (1 канал) | ⚠️ Дублировать на S2 |
| 4+ каналов | ℹ️ Использовать первые 2 |
| Видео > 10 часов | ⚠️ Ограничить 2 часами |
| Недостаточно памяти | ❌ Ошибка с рекомендацией |

## 🏗️ Архитектура проекта

### Модульная структура

```
main.py (entry point)
├── config_loader.py    ← Загрузка YAML конфига
├── ffmpeg_handler.py   ← Извлечение аудиодорожек
├── whisper_handler.py  ← Транскрибация (faster-whisper)
├── dialog_builder.py   ← Сборка диалога из сегментов
├── logger.py           ← JSON логирование
└── errors.py           ← Custom исключения
```

### Workflow (DEMO)

```
config.yaml (video path)
    ↓
1. Load config & validate
2. Extract audio (S1/FL, S2/FR)
3. Transcribe with faster-whisper
4. Merge segments (S1 + S2)
5. Save outputs (txt, srt, json log)
    ↓
output/ (dialog_timeline.txt, .srt, processing.log.json)
```
