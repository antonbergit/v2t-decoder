# 🚀 Phase 4: Deployment в целевом окружении (без Docker)

**Дата**: 28.03.2026  
**Статус**: Принято решение о native/VM deployment

---

## 1) Решение по контейнеризации

В текущем целевом окружении Docker недоступен для рабочего сценария сборки:

- доступ к Docker daemon удалось получить только после добавления пользователя в группу `docker`;
- `docker compose build` падает на ограничениях окружения;
- `docker build` (legacy) также падает из-за ограничений `AppArmor` внутри контейнера.

Итог: **для релиза выбираем native deployment (без Docker)**.

Чтобы не потерять Docker-наработки, они сохранены в отдельной ветке:
- `phase4-docker-poc`

---

## 2) Целевая схема запуска

- Запуск в текущем Linux-окружении (VM/LXC) через Python virtualenv.
- Вход/выход и конфиги — в рабочей директории проекта.
- Базовая команда запуска:

```bash
python3 main.py --config config.yaml
```

## 3) Быстрая установка (native)

### 3.1 Системные пакеты

```bash
sudo apt-get update
sudo apt-get install -y ffmpeg python3 python3-venv python3-pip
```

### 3.2 Python окружение

```bash
cd /home/brgm/v2t-decoder-project/v2t-decoder
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

### 3.3 Проверка

```bash
python -m py_compile main.py src/*.py
python main.py --config config.production-smoke.yaml
```

## 4) Автозапуск через systemd (опционально)

Файл сервиса: [deploy/systemd/v2t-decoder.service](../deploy/systemd/v2t-decoder.service)

Пример установки:

```bash
sudo cp deploy/systemd/v2t-decoder.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now v2t-decoder.service
sudo systemctl status v2t-decoder.service
```
