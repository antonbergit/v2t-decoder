## 2) Что обеспечивает контейнер

- Python 3.11 runtime
- Установку зависимостей из [requirements.txt](../requirements.txt)
- Наличие `ffmpeg` внутри контейнера
- Старт пайплайна через `python main.py --config ...`
- Кеш моделей через volume `whisper-cache`

---

## 3) Быстрый запуск

### Сборка

```bash
docker compose build
```

### Запуск с дефолтным конфигом

```bash
docker compose run --rm v2t-decoder
```

### Запуск с другим конфигом

```bash
CONFIG_FILE=config.production-smoke.yaml docker compose run --rm v2t-decoder
```

## 3.1) Текущая проверка (локально)

- ✅ `docker compose config` проходит (конфиг валиден)
- ⚠️ `docker compose build` в текущей сессии не выполнен из-за прав на Docker socket:
	`permission denied while trying to connect to the docker API at unix:///var/run/docker.sock`

Это инфраструктурное ограничение окружения, а не ошибка Docker-файлов проекта.

## 4) Замечания для ProxmoxVE

1. Для CPU-режима текущей конфигурации достаточно.
2. Для ускорения на GPU потребуется отдельная настройка runtime (NVIDIA Container Toolkit + CUDA image).
3. Видеофайлы и выходные артефакты пробрасываются через bind mount текущего проекта.
